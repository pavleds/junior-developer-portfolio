The problem

Passengers need to answer a few urgent questions: When does the bus leave? Which platform? Is it delayed? A useful station website should make those answers faster to find than asking at an information desk.

This project imagines a modern public-facing website for Vrnjačka Banja station. Because it is an unofficial portfolio project, all schedules, prices, operators, and facilities are explicitly labeled as demonstration data.

The approach

I organized the experience around the departure search instead of filling the homepage with institutional content. The primary action appears immediately, followed by today’s timetable and practical station information.

Scan first

Large times, short route labels, and compact status pills support quick visual scanning.

Filter simply

Destination search and time-of-day filters cover the most common questions without complexity.

Design for movement

The responsive layout prioritizes controls and essential data on narrow screens.

Build trust

Demo notices prevent fictional information from being confused with official schedules.

Technical details

Data-driven rendering

Departures live in a JavaScript array. Filtering produces a smaller array, and the interface is regenerated from that result. This keeps data separate from the HTML structure.

Localization

A translation object stores interface text for Serbian and English. One language function updates labeled elements, placeholders, dates, and results without reloading the page.

Responsive interface

CSS Grid handles the large compositions while media queries reorganize them for tablets and phones. The timetable remains horizontally scrollable when its columns cannot safely fit.

What I learned

  • Time-sensitive interfaces benefit from hierarchy more than decoration.
  • Localization affects dates, placeholders, dynamic status text, and content—not only navigation.
  • Demo projects representing real organizations need visible and repeated disclaimers.
  • Small-screen layouts must be planned as distinct compositions, not simply scaled-down desktop pages.

The outcome

A working bilingual departure finder with search, time filters, service status, station information, and a focused mobile experience. The next production step would be connecting it to an authorized schedule API and content-management system.
